---
title: "Conditionals"
description: "Use Antlers conditionals to show or hide markup."
---

Conditionals let you show markup only when content exists or when a value matches a rule.

## Code usage

### Check a field

Use `{{ if }}` to check if a field has a value.

```html
{{ if hero_image }}
  <img src="{{ hero_image }}" alt="{{ hero_image:alt }}" />
{{ /if }}
```

### Use else

Use `{{ else }}` for a fallback.

```html
{{ if intro_text }}
  <p>{{ intro_text }}</p>
{{ else }}
  <p>{{ excerpt }}</p>
{{ /if }}
```

### Use elseif

Use `{{ elseif }}` for extra branches. Use `elseif`, not `else if`.

```html
{{ if layout == "dark" }}
  <section class="section section--dark">
{{ elseif layout == "light" }}
  <section class="section section--light">
{{ else }}
  <section class="section">
{{ /if }}
```

### Use unless

Use `{{ unless }}` when markup should render only when something is false or empty.

```html
{{ unless hero_image }}
  <div class="image-placeholder"></div>
{{ /unless }}
```

## Basic checks

Check if a value exists.

```html
{{ if title }}
  <h1>{{ title }}</h1>
{{ /if }}
```

Check if a value does not exist.

```html
{{ if ! hero_image }}
  <div class="placeholder"></div>
{{ /if }}
```

## Compare values

Use comparison operators when a field needs to match a specific value.

```html
{{ if layout == "dark" }}
  <section class="section section--dark">
{{ else }}
  <section class="section">
{{ /if }}
```

Common comparisons:

| Operator | Meaning |
| --- | --- |
| `==` | Equals |
| `!=` | Does not equal |
| `>` | Greater than |
| `<` | Less than |
| `>=` | Greater than or equal |
| `<=` | Less than or equal |

## Multiple conditions

Use `&&` when both conditions must be true.

```html
{{ if title && hero_image }}
  <h1>{{ title }}</h1>
  <img src="{{ hero_image }}" alt="{{ hero_image:alt }}" />
{{ /if }}
```

Use `||` when one condition can be true.

```html
{{ if intro_text || excerpt }}
  <p>{{ intro_text ?? excerpt }}</p>
{{ /if }}
```

## Elseif

Use `elseif` for multiple outcomes. This is useful with Replicator sets.

```html
{{ if type == "hero" }}
  {{ partial src="sets.hero" }}
{{ elseif type == "text" }}
  {{ partial src="sets.text" }}
{{ else }}
  {{ partial:if_exists src="sets.{type}" }}
{{ /if }}
```

## Navigation conditionals

Navigation tags expose useful boolean values.

```html
{{ nav:header }}
  <a href="{{ url }}" {{ if is_current || is_parent }}class="current"{{ /if }}>
    {{ title }}
  </a>
{{ /nav:header }}
```

Common navigation conditionals:

| Conditional | Use it for |
| --- | --- |
| `is_current` | The nav item is the current page |
| `is_parent` | The nav item is a parent of the current page |
| `is_external` | The nav item links to an external URL |
| `children` | The nav item has child pages |

Open external links in a new tab.

```html
{{ nav:header }}
  <a href="{{ url }}" {{ if is_external }}target="_blank" rel="noopener"{{ /if }}>
    {{ title }}
  </a>
{{ /nav:header }}
```

Show children only when they exist.

```html
{{ nav:header }}
  <a href="{{ url }}">{{ title }}</a>

  {{ if children }}
    <ul>
      {{ children }}
        <li><a href="{{ url }}">{{ title }}</a></li>
      {{ /children }}
    </ul>
  {{ /if }}
{{ /nav:header }}
```

## No results

Use `no_results` inside tags like `collection` or `nav`.

```html
{{ collection:blog }}
  {{ if no_results }}
    <p>No posts found.</p>
  {{ else }}
    <article>
      <h2><a href="{{ url }}">{{ title }}</a></h2>
    </article>
  {{ /if }}
{{ /collection:blog }}
```

You can also use `unless`.

```html
{{ nav :from="uri" }}
  {{ unless no_results }}
    <a href="{{ url }}">{{ title }}</a>
  {{ /unless }}
{{ /nav }}
```

## Inside loops

Conditionals work inside collection, Grid, and Replicator loops.

```html
{{ collection:projects }}
  {{ if featured }}
    <article>
      <h2>{{ title }}</h2>
      <a href="{{ url }}">View project</a>
    </article>
  {{ /if }}
{{ /collection:projects }}
```

## Current navigation item

Use `is_current` inside navigation loops. Use `is_parent` too when parent menu items should stay active.

```html
{{ nav:header }}
  <a href="{{ url }}" {{ if is_current || is_parent }}class="current"{{ /if }}>
    {{ title }}
  </a>
{{ /nav:header }}
```

## Switch

Use `switch` when inline conditional logic gets long.

```html
{{ switch(
  (layout == "dark") => "section section--dark",
  (layout == "light") => "section section--light",
  () => "section"
) }}
```

## Good defaults

- Check optional fields before rendering markup.
- Use `else` only when the fallback is helpful.
- Use `elseif`, not `else if`.
- Keep complex logic out of templates when possible.
- Use `partial:if_exists` for dynamic partials that may not exist.
