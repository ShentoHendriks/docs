Partials are reusable views. Use them for layout pieces, small components, and Replicator set templates.

## Code usage

### Render a layout partial

Use dot syntax from the recommended `partials` folder. You do not include `partials.` in the tag.

```html
{{ partial:layout.header }}
```

### Render a component partial

```html
{{ partial:components.card }}
```

### Render a dynamic set partial

Use `src` when the partial name is dynamic.

```html
{{ page_builder }}
  {{ partial src="sets.{type}" }}
{{ /page_builder }}
```

When `type` is `hero`, Statamic renders `resources/views/partials/sets/_hero.antlers.html`.

### Render only if the partial exists

Use `partial:if_exists` when a dynamic partial may not exist.

```html
{{ page_builder }}
  {{ partial:if_exists src="sets.{type}" }}
{{ /page_builder }}
```

### Pass data

Pass variables as parameters.

```html
{{ partial:components.card title="Project title" url="/projects/example" }}
```

Inside `_card.antlers.html`, use the parameter handles.

```html _card.antlers.html
<article>
  <h2><a href="{{ url }}">{{ title }}</a></h2>
</article>
```

## Folder usage

| Folder | Use it for | Example |
| --- | --- | --- |
| `layout` | Site structure | `{{ partial:layout.header }}` |
| `components` | Reusable UI pieces | `{{ partial:components.button }}` |
| `sets` | Replicator set templates | `{{ partial src="sets.{type}" }}` |

## Big site structure

For larger sites, split partials by feature area. Keep common pieces shared, and keep feature-specific partials near their purpose.

<Tree>

Use feature partials like this:

```html
{{ partial:collections.projects.card }}
{{ partial:collections.blog.meta }}
{{ partial:forms.newsletter }}
```

Use this structure when shared `components` starts getting too broad. For small sites, keep the simple `layout`, `components`, and `sets` structure.

Partial files start with `_`.

Use `{{ partial:components.card }}` for:

## src vs tag syntax

Use tag syntax for fixed partials.

```html
{{ partial:components.card }}
```

Use `src` for dynamic partial names.

```html
{{ partial src="sets.{type}" }}
```

## Good defaults

- Keep all shared partials inside `resources/views/partials`.
- Prefix partial files with `_`.
- Use `layout` for layout partials.
- Use `components` for reusable UI pieces.
- Use `sets` for Replicator set templates.
- Use `src` only when the partial name is dynamic.
