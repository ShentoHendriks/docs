Collections hold groups of related entries. Use them for pages, blog posts, projects, products, vacancies, or team members.

## Code usage

### List entries

Use the collection handle after `collection:`.

```html
{{ collection:projects sort="title:asc" }}
  <article>
    <h2><a href="{{ url }}">{{ title }}</a></h2>

    {{ if hero_image }}
      <img src="{{ hero_image }}" alt="{{ hero_image:alt }}" />
    {{ /if }}
  </article>
{{ /collection:projects }}
```

Common settings:

| Setting | Example | What it does |
| --- | --- | --- |
| `sort` | `sort="title:asc"` | Sorts by a field handle. Use `:asc` for A-Z or oldest first, and `:desc` for Z-A or newest first. Common examples: `title:asc`, `date:desc`, `order:asc`. |
| `limit` | `limit="6"` | Shows only the first number of matching entries. Use it for sections like "latest 3 posts" or "featured 6 projects". |
| `offset` | `offset="3"` | Skips entries before rendering. Use it when one entry is already shown somewhere else, like a featured card above a grid. |

Common `sort` values:

| Sort value | Use it for |
| --- | --- |
| `title:asc` | Alphabetical lists, like team members or projects |
| `title:desc` | Reverse alphabetical lists |
| `date:desc` | Newest first, like blog posts or news |
| `date:asc` | Oldest first, like timelines or event archives |
| `order:asc` | Manual order from the control panel |
| `order:desc` | Reverse manual order |

### Use from

Use `from` when the collection handle should be a parameter.

```html
{{ collection from="projects" sort="title:asc" }}
  <article>
    <h2><a href="{{ url }}">{{ title }}</a></h2>
  </article>
{{ /collection }}
```

Common settings:

| Setting | Example | What it does |
| --- | --- | --- |
| `from` | `from="projects"` | Chooses the collection |
| `sort` | `sort="title:asc"` | Sorts entries by field and direction |
| `limit` | `limit="6"` | Limits how many entries render |

Use a variable inside `from` when editors choose the collection.

```html
{{ collection from="{selected_collection}" limit="6" }}
  <article>
    <h2><a href="{{ url }}">{{ title }}</a></h2>
  </article>
{{ /collection }}
```

Common dynamic settings:

| Setting | Example | What it does |
| --- | --- | --- |
| `from` | `from="{selected_collection}"` | Uses a field or variable as the collection handle |
| `limit` | `limit="6"` | Limits how many entries render |

<Note>
  Use `from` for collections. Use `src` for partials.
</Note>

<Accordion title="Multiple collections">

Separate multiple collection handles with `|`.

```html
{{ collection from="blog|events" sort="date:desc" limit="6" }}
  <article>
    <h2><a href="{{ url }}">{{ title }}</a></h2>
  </article>
{{ /collection }}
```

Common settings:

| Setting | Example | What it does |
| --- | --- | --- |
| `from` | `from="blog|events"` | Pulls entries from more than one collection |
| `sort` | `sort="date:desc"` | Shows newest dated entries first |
| `limit` | `limit="6"` | Limits how many entries render |

</Accordion>

<Accordion title="Alias results with as">

Use `as` when you need wrapper markup around the results.

```html
{{ collection from="blog" as="posts" }}
  {{ if no_results }}
    <p>No posts found.</p>
  {{ else }}
    <ul>
      {{ posts }}
        <li><a href="{{ url }}">{{ title }}</a></li>
      {{ /posts }}
    </ul>
  {{ /if }}
{{ /collection }}
```

Common settings:

| Setting | Example | What it does |
| --- | --- | --- |
| `from` | `from="blog"` | Chooses the collection |
| `as` | `as="posts"` | Stores results under a custom variable |
| `no_results` | `{{ if no_results }}` | Checks if the query returned nothing |

</Accordion>

<Accordion title="Scope fields">

Use `scope` when entry fields conflict with fields already on the page.

```html
{{ collection from="blog" scope="post" }}
  <article>
    <h2><a href="{{ post:url }}">{{ post:title }}</a></h2>
  </article>
{{ /collection }}
```

Common settings:

| Setting | Example | What it does |
| --- | --- | --- |
| `from` | `from="blog"` | Chooses the collection |
| `scope` | `scope="post"` | Prefixes entry variables with `post:` |

</Accordion>

<Accordion title="Show one entry">

On a single entry template, use fields directly. You do not need the collection tag.

```html
<article>
  <h1>{{ title }}</h1>
  {{ content }}
</article>
```

</Accordion>

## When to use collections

Use a collection when editors create multiple items with the same content type.

Common examples:

| Collection | Use it for |
| --- | --- |
| `pages` | Normal website pages |
| `blog` | News or articles |
| `projects` | Portfolio or case studies |
| `team` | Team member profiles |
| `products` | Product pages |
| `vacancies` | Job posts |

## Create a collection

Create collections in the control panel.

1. Open **Collections**.
2. Click **Create collection**.
3. Enter a name like `Pages`, `Blog`, or `Projects`.
4. Choose whether entries need URLs.
5. Set the route.
6. Set the template.
7. Save the collection.

## Routes

The **Route** controls each entry URL. It is a pattern with variables inside curly braces.

### Basic routes

| Use case | Route | Example URL |
| --- | --- | --- |
| Pages | `{slug}` | `/about` |
| Blog posts | `blog/{slug}` | `/blog/my-first-post` |
| Projects | `projects/{slug}` | `/projects/site-redesign` |

With `blog/{slug}`, an entry titled `My first post` becomes `/blog/my-first-post`.

With `{slug}`, an entry titled `About` becomes `/about`.

### Route variables

| Variable | Use it for | Example |
| --- | --- | --- |
| `{slug}` | Entry slug | `blog/{slug}` |
| `{year}` | Dated collections | `blog/{year}/{slug}` |
| `{month}` | Dated collections | `blog/{year}/{month}/{slug}` |
| `{day}` | Dated collections | `blog/{year}/{month}/{day}/{slug}` |
| `{parent_uri}` | Nested entries | `{parent_uri}/{slug}` |
| `{mount}` | Mounted collections | `{mount}/{slug}` |

You can also use custom entry fields in routes. Keep this simple unless the project needs advanced routing.

<Tip>
  A route like `blog/{slug}` creates URLs for blog entries. It does not create the `/blog` listing page.
</Tip>

## Index and show templates

Scaffolding usually gives you two templates:

<Tree>
  <Tree.Folder defaultOpen name="resources">
    <Tree.Folder defaultOpen name="views">
      <Tree.Folder defaultOpen name="blog">
        <Tree.File name="index.antlers.html" />
        <Tree.File name="show.antlers.html" />
      </Tree.Folder>
    </Tree.Folder>
  </Tree.Folder>
</Tree>

### index

Use `index.antlers.html` for the listing page.

Example URL: `/blog`

```html
{{ collection:blog sort="date:desc" }}
  <article>
    <h2><a href="{{ url }}">{{ title }}</a></h2>
    {{ if excerpt }}
      <p>{{ excerpt }}</p>
    {{ /if }}
  </article>
{{ /collection:blog }}
```

### show

Use `show.antlers.html` for one entry detail page.

Example URL: `/blog/my-first-post`

```html
<article>
  <h1>{{ title }}</h1>
  {{ content }}
</article>
```

## Make a listing page

The `index.antlers.html` file is only a template. It does not create `/blog` by itself.

Recommended GUI setup:

1. Create a `Pages` collection if you do not have one.
2. Create a page entry named `Blog`.
3. Set its slug to `blog`.
4. Set its template to `blog/index`.
5. In the `Blog` collection settings, set the entry template to `blog/show`.

Result:

| URL | Template | Purpose |
| --- | --- | --- |
| `/blog` | `blog/index` | Lists blog entries |
| `/blog/my-first-post` | `blog/show` | Shows one blog entry |

## Entry fields

Every entry has fields from its blueprint. Use the field handle in templates.

Example page fields:

| Display | Handle | Code usage |
| --- | --- | --- |
| Hero title | `hero_title` | `{{ hero_title }}` |
| Hero image | `hero_image` | `{{ hero_image }}` |
| SEO title | `seo_title` | `{{ seo_title }}` |

## Stored files

Statamic saves collection config and entries as project files. Editors do not need to edit these files directly.

<Tree>
  <Tree.Folder defaultOpen name="content">

      <Tree.File name="pages.yaml" />
      <Tree.Folder defaultOpen name="pages">
        <Tree.File name="home.md" />
        <Tree.File name="about.md" />
        <Tree.File name="contact.md" />
      </Tree.Folder>
    </Tree.Folder>
  </Tree.Folder>
</Tree>

## Good defaults

- Use plural handles: `projects`, not `project`.
- Keep handles lowercase and use underscores if needed.
- Use `pages` for normal site pages.
- Use dedicated collections for structured content like projects or team members.
- Use `from` when the collection handle comes from a variable.
