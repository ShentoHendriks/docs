Use the Statamic control panel for editing content, collections, globals, fields, and blueprints. Treat saved project files as developer-only storage.

## Code usage

Render a page with collection fields, global content, and a Replicator page builder.

```html
<main>
  <h1>{{ title }}</h1>

  {{ page_builder }}
    {{ partial src="sets.{type}" }}
  {{ /page_builder }}
</main>

<footer>
  <a href="mailto:{{ company:email }}">{{ company:email }}</a>
</footer>
```

## Quick decision guide

Use a **Collection** when editors create multiple items of the same type.

Use **Globals** when content belongs to the whole site, not one URL.

Use a **Blueprint** when you need to control which fields editors see.

Use **Replicator** when editors need flexible ordered blocks, like hero, text, image, CTA, and FAQ sections.

Use **Grid** when editors need repeated rows with the same columns, like pricing rows, feature lists, or team contact details.
