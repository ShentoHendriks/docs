## Overview

Use generated Rainy Weather Framework utilities in production builds through Tailwind.

Make sure the generated CSS is included in your application before deploying.

## Build Check

Run your normal production build:

```bash
npm run build
```

Then verify pages that use framework utilities still render expected layout, spacing, typography, and colors.

## What To Check

Before deploying, check:

- text style utilities such as `display-l`, `heading-h2`, and `body-m`
- spacing utilities such as `py-section`, `mt-xl`, and `gap-lg`
- color utilities such as `bg-background-default` and `text-text-default`
- container utilities such as `container-grid`, `container-grid-fluid`, and `container-grid-fullscreen`
- column utilities such as `w-col-*`, `ml-col-*`, `pl-col-*`, and `gap-col-*`
- responsive variants such as `switch:` and `desktop:`

## Deployment

Deploy the compiled application, not the source utilities alone.

The production bundle should include the generated theme tokens and utilities used by your templates.
