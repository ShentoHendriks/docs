## Overview

Text style utilities are role-based typography classes.

Examples:

```text
display-l
heading-h1
heading-h2
body-l
body-m
label-m
caption-m
button-m
```

They are generated as Tailwind `@utility` classes.

```html
<h1 class="display-l">Main headline</h1>
<p class="body-l">Lead paragraph</p>
```

## What They Include

A text style can include:

- font family
- font size token
- line height
- letter spacing
- font weight
- font style
- text transform

Example output shape:

```postcss
@utility display-l {
  font-family: var(--font-sans);
  font-size: var(--text-7xl);
  line-height: 0.95;
  letter-spacing: 0;
  font-weight: 700;
  font-style: normal;
  text-transform: none;
}
```

## Why Use Text Styles

**Text tokens** describe size.

**Text styles** describe purpose.

Less ideal:

```html
<h1 class="text-7xl font-bold leading-none">
  Headline
</h1>
```

Better:

```html
<h1 class="display-l">
  Headline
</h1>
```

`display-l` carries design intent. It also stays synced with Figma text styles.

## Usage

```html
<section class="container-grid py-section-xl">
  <p class="label-m mb-md">New Release</p>

  <h1 class="display-l">
    A complete fluid design system.
  </h1>

  <p class="body-l mt-lg w-col-6">
    Text styles keep typography consistent across design and code.
  </p>
</section>
```

## Fluid Behavior

Text styles use generated typography tokens, usually `clamp(...)`.

That means text grows between configured min and max viewport widths.

Inside `container-grid-fluid`, text styles can receive special overrides above max viewport, so display typography keeps scaling with very wide layouts.

```html
<section class="container-grid-fluid">
  <h1 class="display-l">Scales wider on large screens.</h1>
</section>
```

Inside `container-grid` and `container-grid-fullscreen`, text styles keep normal capped `clamp(...)` behavior.

## Best Practice

Use **text styles** for real content.

Use raw `text-*` classes only for exceptions or one-off technical cases.
